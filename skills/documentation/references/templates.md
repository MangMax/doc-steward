# Documentation Templates

Use these only when a project needs the corresponding document and the content can be grounded in verified project evidence. Remove unused sections, blank rows, and placeholder text before finishing.

## Architecture

```md
# Architecture

## Summary

Briefly describe the system structure.

## Major Components

| Component | Responsibility |
|---|---|
|  |  |

## Data Flow

Describe the main flow of data or control.

## Key Dependencies

List important dependencies and why they exist.

## Boundaries

Explain what each major module should and should not do.

## Tradeoffs

Document important compromises.

## Open Questions

List unresolved architectural questions.
```
